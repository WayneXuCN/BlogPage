## 前言

工欲善其事，必先利其器。合适的工具能够大大提升科研效率。以下是我在日常工作中经常使用的工具清单。

## 1. 文献管理 - Zotero

> [!NOTE]
> Zotero 是我目前主力使用的文献管理工具，开源免费且功能强大。

**优点：**

- 浏览器插件一键保存文献
- 自动提取元数据
- 与 Word/Google Docs 无缝集成
- 支持 WebDAV 同步

**推荐插件：**

- Zotero Connector（浏览器插件）
- Better BibTeX（导出 BibTeX）
- ZotFile（PDF 管理）

## 2. 写作工具 - Obsidian

Obsidian 是基于本地 Markdown 文件的知识管理工具。

**为什么选择 Obsidian：**

- 双向链接，构建知识网络
- 本地优先，数据在自己手中
- 插件生态丰富
- 支持 LaTeX 公式

```markdown
# 我的使用场景

- 日常笔记
- 文献笔记（配合 Zotero）
- 项目管理
- 博客草稿
```

## 3. 代码编辑 - VS Code

VS Code + 一系列插件是我的主力开发环境。

**必装插件：**

- Python / Pylance
- GitHub Copilot（AI 辅助）
- GitLens
- Jupyter
- Remote SSH

## 4. 数据分析 - JupyterLab

对于交互式数据分析，JupyterLab 依然是首选。

```python
# 我的常用启动配置
jupyter lab --no-browser --port=8888
```

## 5. 版本控制 - Git + GitHub

无论是代码还是论文，版本控制都是必须的。

**工作流：**

1. 每次修改前创建分支
2. 频繁提交，写清楚 commit message
3. 使用 Pull Request 进行代码审查

## 6. 任务管理 - Todoist

简洁高效的任务管理工具。

**我的使用方式：**

- 按项目分类
- 设置优先级和截止日期
- 每日回顾

## 7. 终端工具 - Warp

Warp 是现代化的终端工具，支持 AI 命令补全。

**亮点功能：**

- 命令块（每条命令结果独立显示）
- AI 命令建议
- 工作流自动化

## 8. 截图与标注 - CleanShot X

macOS 上最好用的截图工具。

## 9. API 测试 - Bruno

Bruno 是 Postman 的开源替代品，配置文件存储在本地。

## 10. 浏览器 - Arc

重新定义浏览器体验。

**特色功能：**

- Space 功能分组标签
- 侧边栏笔记
- Boost 自定义网站样式

## 总结

选择工具的原则：

1. **开源优先** - 数据自主可控
2. **简洁高效** - 功能够用就好
3. **可定制性** - 适应个人工作流
4. **跨平台** - 多设备无缝切换
